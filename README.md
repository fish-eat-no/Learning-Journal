# Learning-Journal
记录每日学习内容
第一部分：手把手教你搭建学习日志
1. 创建一个专门的仓库
不用把日志散落在各种项目里，单独建一个仓库专门放学习记录。
• 仓库名称建议：Learning-Journal 或 CS-Learning-Notes（简洁明了就行）
• 可见性：建议设为 Public（公开）。这样以后找实习时，可以直接把链接贴在简历里 。如果暂时不想公开，也可以先选 Private。
2. 仓库结构规划（二选一）
计算机学习内容比较杂，有算法、有课程笔记、有小项目。推荐两种组织方式，你可以根据自己的习惯选：
• 按时间线组织（适合每天坚持记录）：
text
CS-Learning-Notes/
├── README.md          # 仓库总介绍，放学习进度表
├── 2025-02-24/        # 以当天日期建文件夹
│   ├── README.md      # 今日学习日志
│   └── algorithm/     # 今天刷的算法题代码
├── 2025-02-25/
│   ├── README.md
│   └── os-note.md     # 今天看的操作系统笔记
• 按主题组织（适合集中攻破某个方向）：
text
CS-Learning-Notes/
├── README.md
├── Algorithms/        # 算法专区
│   ├── 排序/
│   └── 动态规划/
├── Backend/           # 后端学习
│   ├── 数据库/
│   └── Redis/
└── Projects/          # 练手小项目
3. 核心操作流程
1. 初始化仓库：在 GitHub 网页端创建好后，git clone 到本地。
2. 每天记录：在对应目录下创建当天的 Markdown 文件（如 2025-02-24.md）。
3. 提交更改：写完之后，用 Git 提交 ：
bash
git add .
git commit -m "feat: 完成二叉树层序遍历学习"
git push origin main
小建议：提交信息写清楚，方便以后翻日志 。
4. 几个能让 GitHub 日志更好用的小功能
• 用 Issues 记录待办和问题：可以把学习中遇到的 Bug、没搞懂的概念记在 Issues 里，打上 question、bug 标签 。
• 用 Projects 做进度看板：GitHub 自带的 Projects 功能可以做成 To do / In progress / Done 三栏，适合跟踪学习计划 。
• 善用 .gitignore：如果日志里放了代码，记得忽略 node_modules、.idea 这类文件 。

建议每天新建一个文件，文件名用 YYYY-MM-DD.md。内容结构可以这样 ：

## 📅 今日学习目标
- [ ] 完成《算法导论》第 12 章二叉搜索树
- [ ] LeetCode 刷题 3 道（树相关）
- [ ] 整理操作系统进程部分笔记
## 📚 学习内容
### 1. 二叉搜索树
- 理解了前驱、后继节点的查找逻辑
- 删除操作比较绕，分三种情况讨论：无子节点、单子节点、双子节点
### 2. LeetCode 刷题记录
| 题号 | 题目 | 思路 | 代码 |
|------|------|------|------|
| 98 | 验证二叉搜索树 | 中序遍历是否递增 | [链接](./code/98.py) |
| 230 | 二叉搜索树中第K小的元素 | 中序遍历计数 | [链接](./code/230.py) |
## 💡 重点/难点记录
**二叉搜索树删除节点**（双子节点情况）：
- 找到右子树的最小节点（或左子树的最大节点）
- 替换当前节点值
- 递归删除那个用于替换的节点
## ⚠️ 遇到的问题
- 递归删除时忘记处理父节点指针，导致出现悬空引用
- ✅ 解决方案：画图模拟了一遍流程，发现需要用返回值接收新子树
## 📝 总结与复盘
- 树的递归问题，关键在于相信函数定义，不要试图在脑子里压栈
- 明天开始看 AVL 树的旋转
